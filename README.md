# ShpoolkenDB

База данных филаментов для 3D-печати, вдохновлённая [SpoolmanDB](https://github.com/donkie/SpoolmanDB) и созданная, чтобы исправить его главную проблему — полное отсутствие поддержки.

## Проблема

Оригинальный SpoolmanDB — отличная идея, хорошая задумка, и... всё. На практике проект стоит на месте: сотни филаментов не добавлены, PR лежат месяцами, а на issue никто не отвечает. Хочешь добавить новый цвет от Bambu Lab? Открывай PR и жди. Может, кто-то посмотрит. Когда-нибудь.

Сообщество 3D-печати ждало. И ждало. И ждало.

## Решение

ShpoolkenDB появился потому, что кому-то это надоело. Это проект с миссией: действительно поддерживать базу данных филаментов. Новые филаменты добавляются. Обновления выходят. На issues отвечают. Революционная идея, да.

База хостится на GitHub Pages: [https://annenskei.github.io/ShpoolkenDB/](https://annenskei.github.io/ShpoolkenDB/)

## Структура

- **filaments/** — JSON-файлы для каждого производителя. Добавить или обновить филамент? Редактируй нужный файл.
- **materials.json** — Общие свойства материалов (PLA, PETG, ABS и т.д.).
- **Компиляция** — При деплое всё собирается в единый `filaments.json`.

## Вклад

Да, именно тебя мы и ждали.

1. Форкни репозиторий.
2. Отредактируй или добавь файл производителя в `filaments/`.
3. Отправь PR.
4. Получи ответ. На этот раз — реально.

## Поля исходных файлов

- **name** — Название продукта. Используй `{color_name}` для автоподстановки цвета.
- **material** — Тип материала (PLA, PETG, ABS и т.д.).
- **density** — Плотность материала в г/см³.
- **weights** — Массив объектов с полями `weight`, `spool_weight` и `spool_type` ("plastic", "cardboard", "metal").
- **diameters** — Массив диаметров в мм (например, `1.75`, `2.85`).
- **extruder_temp** *(опционально)* — Рекомендуемая температура экструдера в °C.
- **bed_temp** *(опционально)* — Рекомендуемая температура стола в °C.
- **finish** *(опционально)* — Покрытие филамента: "matte", "glossy" и т.д.
- **multi_color_direction** *(опционально)* — "coaxial" для разделённого/двухцветного филамента, "longitudinal" для градиента вдоль длины.
- **pattern** *(опционально)* — "marble" или "sparkle" (или предлагай свои).
- **translucent** *(опционально)* — `true`/`false`.
- **glow** *(опционально)* — `true`/`false` для светящихся в темноте филаментов.
- **colors** — Массив объектов с полями `name` и `hex`. Для мультиколора используй `hexes` (массив) вместо `hex`.
